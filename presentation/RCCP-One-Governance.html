The monthly RCCP cycle

One validated plan per month: inputs in → build & validate → stakeholder review → sign-off. Everyone debates the same published numbers.

1

Inputs

Working days 1–3 · data owners
  • S&OP — demand forecast
  • Product — portfolio / SKU changes
  • Manufacturing — line capacity, OEE & headcount
  • Warehouse — pallet capacity PHASE 2
  • Standing data — master data, as needed (Planning + Eng.)
→
2

Build & validate

Working days 4–6 · Planning
  • Create the batch, upload the SAP files
  • Run the 7-stage validation — fix at source
  • Publish a candidate baseline (single source of truth)
→
3

RCCP review

Mid-month · stakeholders
  • Chaired by Planning; S&OP, Manufacturing, Product, Warehouse, Finance
  • Review capacity vs order book, risk lines & headcount
  • Run scenarios — extra hours / OEE / £ cost
  • Agree actions & owners
→
4

Sign-off

By month-end · approvers
  • ELT / Site Ops — approve the plan
  • Finance — approve any extra cost PHASE 2
  • Baseline locked & published — the agreed plan
↻

If the review needs changes: Planning corrects the source file → re-validates → re-publishes a new candidate. Every version is kept — full audit trail.
